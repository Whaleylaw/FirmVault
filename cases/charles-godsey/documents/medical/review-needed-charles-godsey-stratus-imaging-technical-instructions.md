Status Descpription
Failed The exam failed to be pushed.
1. Click the caret icon to display more options.
2. Click Push Again to try to re-push the exam.
3. Click See Failure Details to see the transmitted error message.
The following errors may be listed:
Failed to establish association: Could not find a PACS connection.
Hex number followed by a status - for example, "0x0110: Unknown Status Code":
This indicates the PACS returned this code when it failed to receive the image. This
needs to be investigated with the client's PACS system. For instance, this is
commonly a result of the client PACS refusing structured reports or presentation
states (GSPS, etc.).
DIMSE No valid Presentation Context ID: The PACS responded that it doesn't
support the images being sent - - specifically the combination of SOP class UID and
transfer syntax. This specific image would need to be investigated further with the
client to determine if the DICOM is incorrect or if it is being rejected by the PACS.
Not All Instances Sent: This is usually coupled with another error above. If it occurs
alone, this needs to be investigated in the DicomStoreService logs on the edge
server sending the images to see what the actual error is being returned.
Etimedout: This indicates the download of the image from the cloud failed and
needs to be investigated in the DicomStoreService logs. Normally the resolution to
this issue is to simply re-push the images to PACS. If the cloud-side is known to be
healthy and the re-pushing the exam still returns this error, this should be escalated
to development

exams to upload. An administrator can also configure the workflow so that users can modify patient information

Mode).
