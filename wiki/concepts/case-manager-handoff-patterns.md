# Case Manager Handoff Patterns

> PI cases frequently outlast the tenure of the case manager who initially worked them. When a case manager leaves or cases are reassigned, the incoming staff member must pick up an unfamiliar file, establish rapport with the client, and continue without losing momentum. These handoffs create a risk of dropped tasks, communication gaps, and client frustration.

## Pattern

### Handoff Triggers
1. **Staff turnover**: A case manager leaves the firm and their cases are redistributed
2. **Workload rebalancing**: Cases are reassigned for caseload management
3. **Specialization**: A case moves to a phase requiring a different skill set (e.g., from treatment monitoring to demand preparation)

### Observable Signals of Handoff
- Change in the staff name on emails and call logs (e.g., correspondence shifts from one paralegal to another)
- New case manager may need to request basic information already in the file (e.g., which franchise location the client worked at)
- Communication gaps during the transition period
- Client may not be formally introduced to the new case manager

### Handoff Risks
1. **Knowledge loss**: The outgoing case manager's institutional knowledge of the case (adjuster relationship, client preferences, pending issues) is not fully transferred
2. **Task gaps**: Pending follow-ups may be dropped during the transition
3. **Client confusion**: The client suddenly hears from a different person without explanation
4. **Duplicate work**: The new case manager may re-request information already obtained

### Best Practice (Observed)
- Some cases show smooth handoffs where the new case manager CCs the prior one during the transition period
- Filevine project email addresses maintain continuity regardless of case manager changes
- The new case manager should review the full activity log before first client contact

## Evidence

- In a workplace injury case, the initial case manager handled the file from intake (2/2024) through approximately mid-2024. A second paralegal took over and continued through 2026. The handoff was visible in the activity log as the corresponding name on emails changed. The new case manager had to ask the client basic questions already answered in the file (e.g., the specific location where they worked), suggesting incomplete knowledge transfer.
- In multiple cases, the same paralegal (Sarena) handled the case from intake through years of treatment/litigation, only for a different staff member (Bryce) to take over later phases. The CC pattern during transition helped maintain continuity.
- In Case M (MVA, DOI 8/2023), the case was touched by at least 4 staff members over its lifecycle: one attorney handled intake and initial client contact, a paralegal handled file setup and records requests, then the lead attorney managed demand and negotiation, and finally a different attorney took over litigation (depositions, mediation scheduling). The handoff from negotiation to litigation coincided with the case moving from inter-counsel policy distribution to formal suit. The client's frustration ("does not receive call backs or is told conflicting information repeatedly") was explicitly attributed to multiple staff giving different information, confirming that handoff-related inconsistency is a direct client frustration trigger.
- In Case N (premises liability estate, DOI 7/2020), the case spanned 5+ years and was touched by multiple staff. The paralegal (Sarena) handled litigation tasks (discovery responses, filing) while attorneys handled case strategy. The case was referred out to outside counsel (Joel Franklin) at one point, then continued with the original firm. The surviving spouse's repeated calls for updates that went unanswered suggest communication gaps during the handoff periods. This confirms that 5+ year cases will inevitably experience multiple handoffs and each one creates a communication risk.

## Agent Guidance

1. When taking over a case, read the last 3 months of activity logs before contacting anyone
2. Send an introduction email to the client: "My name is [X], I'll be working on your case going forward"
3. Review all pending tasks and follow-up dates — assume nothing was done during the gap
4. Check for any adjuster contacts that need to be re-engaged
5. Verify the case's current phase and landmarks against what's actually been completed
6. Do not ask the client for information already documented in the file
7. If possible, have the outgoing case manager provide a verbal briefing on each case

## Pitfalls

- Not reading the case history before contacting the client or adjuster
- Asking the client for information already in the file (damages credibility)
- Dropping pending follow-ups during the transition
- Not introducing yourself to the client (they may not know who you are)
- Losing track of the adjuster relationship built by the prior case manager
